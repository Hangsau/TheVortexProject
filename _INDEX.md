# The Vortex Project — 系統索引

> 核心命題：所有游泳技術都建立在水感知上  
> 技術形式是感知系統的輸出，不是輸入  
> 作者：Hangsau · 建立：2026

---

## 系統架構

```
TheVortexProject/
│
├── Instructional/       物理層：四式技術的物理條件與生物力學
├── Technica/            感知層：L0–L6 水感發展框架
├── Bridge/              橋接層：每個技術點的感知目標語言
├── Research/            研究層：文獻筆記與前沿假設
│   ├── 感知科學/
│   └── 物理現象/
└── Observations/        觀察層：教學案例記錄
```

### 各層說明

**Instructional/** — 為什麼這樣游才快  
技術的物理條件：EVF、旋轉連動、渦流利用、疲勞崩潰序列  
文件格式：.md（原 .docx 保留備份）

**Technica/** — 如何建立感知  
L0–L6 水感發展框架、三型診斷（A/B/C）、技術指標對應  
文件格式：.md（原 .docx 保留備份）

**Bridge/** — 兩層之間的橋  
每個技術點對應的感知目標語言  
格式：「物理描述 → 感知目標 → 感知失敗的樣子」  
文件格式：.md，方便快速更新

**Research/** — 文獻與假設  
感知科學基礎（機械感受器、feedforward、感知學習）  
物理現象探索（渦流回收、自然頻率、彈性蓄能）  
文件格式：.md，每個主題一份

**Observations/** — 教學現場  
學生案例記錄、感知突破事件、三型診斷驗證  
文件格式：.md，每個學生一份
