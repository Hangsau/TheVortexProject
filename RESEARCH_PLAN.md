# The Vortex Project — 研究規劃架構

> 建立日期：2026-04-10  
> 性質：個人教學研究 × 純粹好奇驅動  
> 核心命題：**所有技術都建立在水感知上**  
> 年限：不設限，依好奇心與教學需求推進

---

## 研究架構總覽

這個計劃分四個研究層，每層有不同的認識論性質：

```
Layer 0 │ 系統完整性      → 把已知的東西做完、做紮實
Layer 1 │ 感知科學基礎    → 理解「水感知」的底層機制
Layer 2 │ 教學觀察研究    → 在真實教學中驗證與發現
Layer 3 │ 物理現象探索    → 探索還沒人問過的問題
        │
        ↓
整合層  │ 持續把發現寫回系統
```

四層不是線性推進，是**同時存在的研究場域**，每次學習或教學都可能在不同層產生進展。

## Layer 1 — 感知科學基礎

**性質：理解「水感知」的底層機制**  
**方法：文獻閱讀與理論建構**  
**問題：水感知是什麼？它如何建立？它的上限在哪裡？**

這一層的目標不是「做研究」，而是把神經科學、感知心理學、運動學習的既有知識，系統地整合進這個框架。

### 1.2 Feedforward 固化機制 ✅ 完成（2026-04-18）

`Research/感知科學/feedforward機制.md`

## Layer 2 — 教學觀察研究

**性質：在真實教學中觀察、記錄、找模式**  
**方法：系統化案例記錄**  
**問題：這套框架在真實教學中的行為是什麼？**

這一層不需要實驗設計，需要的是**系統化的觀察習慣**。

### 2.1 建立觀察記錄系統

每個學生建立一份觀察檔案，記錄：

```
基本資料
├── 游泳年資、教學背景
├── 入學時的感知層次評估（L 級）
├── 三型診斷結果（A/B/C）
└── 解剖邊界評估（踝關節、髖內旋、肩活動度）

訓練記錄
├── 每次訓練的主要訓練方法
├── 學生的感知描述語言（他們怎麼描述自己的感覺）
├── 技術變化的外部觀察指標
└── 瓶頸點：卡在哪裡、卡多久

進展追蹤
├── L 級提升的觸發事件（是什麼讓他們突破的？）
├── 三型診斷的變化（有沒有從一型轉到另一型？）
└── 技術在壓力/疲勞下的穩定性
```

**現況：** `Observations/` 已有三份案例（L0漂浮三態診斷、小划手板實驗、指令層次設計），方向正確，持續累積。

### 2.2 核心觀察問題

這些是有答案後，整個框架會變得更紮實的問題：

**關於三型診斷：**
- A型、B型、C型——這個分類在教學現場的預測力有多準？
- 一個學生被診斷為 A型後，使用 A型訓練方法，進展速度比沒有分型快嗎？
- 有沒有第四型的存在？（目前框架是否有未被涵蓋的感知缺陷模式？）

**關於感知剝奪訓練：**
- 「復活感」（脫掉干擾工具後的高感知窗口）平均持續多久？
- 在哪個 L 級，感知剝奪訓練效果最顯著？
- 拳頭游 vs 穿襪踢水 vs 其他剝奪工具，效果有可比較的差異嗎？

**關於感知與技術穩定性：**
- 感知優先訓練的學生，在疲勞下的技術崩潰序列，跟形式優先訓練的學生有什麼不同？
- 一個技術動作「正確但感知未建立」和「感知已建立但形式還不穩定」，哪個在壓力下更穩健？

## 研究的態度原則

**區分確定性層次**（這套系統已在做，要繼續保持）

```
🔵 邏輯推導    — 物理或生理學演繹，原則上跨時間有效
🟢 近期文獻    — 有實證支持，但有信賴區間
🟡 有效舊文獻  — 物理原理仍成立，具體數值可能已更新
🟠 教練觀測    — 有實務參考價值，非學術控制組
🔴 個人假設    — 目前是直覺，需要更多文獻或觀察支持
```

**研究不是為了得到「正確答案」**

每個層次的研究，都可能讓你發現：
- 現有框架的某個假設是錯的
- 某個現象比想像的複雜
- 某個被認為重要的東西，實際上沒那麼重要

這些發現和「確認假設」同樣有價值。
